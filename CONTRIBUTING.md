# Contributing

Thank you for your interest in contributing! All contributors must follow the **[Code of Conduct](./CODE_OF_CONDUCT.md)**.

## 1. Branching

* Features, bug fixes, and experiments should have their **own branches**.
* Naming convention:

  ```
  feature/cool-feature
  fix/issue-description
  ```

* Always branch off `main` (or the latest stable branch)..

## 2. Commits

* Make **small, focused commits**.
* Use clear, descriptive messages:

  ```
  [module] Short summary of change
  ```

* Example:

  ```
  [commons-bukkit] Implemented Login logic
  ```

## 3. Pull Requests

* PRs must be **descriptive**, explaining what was changed and why.
* Include related issue references if applicable.
* PRs will be reviewed for:

  * Code quality
  * Compliance with coding standards

## 4. Reporting Issues

* Use GitHub Issues for bugs or enhancements.
* Provide:

  * Steps to reproduce
  * Expected vs. actual behavior
  * Relevant logs or stack traces
